## 二、数据查询题（10 小题，每小题 4 分，共 40 分）

设有关系模式 `Department`、`Student`、`Course`、`CourseRel` 和 `SC`，结构如下：

```text
Department(dNo, dName, addr, tel)
```

其中，`dNo`、`dName`、`addr`、`tel` 分别表示学院代号、学院名称、办公地址、电话号码；`dNo` 为主码。

```text
Student(sNo, sName, sex, age, dNo)
```

其中，`sNo`、`sName`、`sex`、`age`、`dNo` 分别表示学号、姓名、性别、年龄、所属学院代号；`sNo` 为主码，`dNo` 为外码，参照 `Department(dNo)`。

```text
Course(cNo, cName, type, credit, dNo)
```

其中，`cNo`、`cName`、`type`、`credit`、`dNo` 分别表示课程号、课程名、课程类别、学分、开课学院代号；`cNo` 为主码，`dNo` 为外码，参照 `Department(dNo)`。

```text
CourseRel(cNo, cPNo)
```

其中，`cNo`、`cPNo` 分别表示课程号、先修课程号；`cNo`、`cPNo` 联合为主码，均为外码，参照 `Course(cNo)`。

```text
SC(sNo, cNo, score)
```

其中，`score` 表示课程成绩；`sNo`、`cNo` 联合为主码，`sNo` 为外码，参照 `Student(sNo)`，`cNo` 为外码，参照 `Course(cNo)`。

### 1. 使用关系代数表达式完成以下查询

（1）查询“软件学院”开设的学分大于等于 3 的所有课程名称。

（2）查询选修了“数据库系统”课程但没有选修“操作系统”课程的所有学生姓名。

（3）查询选修了“软件学院”开设的全部课程的学生学号和姓名。

### 2. 使用 SQL 语句完成以下操作

> 本部分不使用 `EXISTS` / `NOT EXISTS`，也不使用 SQL 中的 `UNION`、`INTERSECT`、`EXCEPT` 等集合操作。

（4）若关系 `SC` 在创建时没有添加主码和外码，请在关系 `SC` 创建后通过 SQL 为其添加主码和外码。

（5）创建一个学生成绩视图 `v_sw_student_avg`，包含软件学院每名有选课记录学生的学号、姓名和所有已录入成绩的平均分。

（6）将“李明”同学的“数据库系统”课程成绩增加 10 分，但增加后的成绩最高不超过 100 分。

（7）查询选修并且考核通过（成绩大于等于 60 分）的课程总学分大于等于 20 的所有学生姓名。

（8）查询“数据库系统”课程的所有直接先修课程名称。

（9）查询只选修了“算法”一门课程的学生学号和姓名。

（10）查询所有具有不及格课程的学生姓名、课程名和成绩。其中，成绩小于 60 分或成绩为空均视为不及格。

## 三、数据库设计题（20 分）

某高校运动会竞赛管理信息系统需要保存如下信息：

1. 学院信息包括学院编号、学院名称、办公地址、联系电话。
2. 学生运动员信息包括学号、姓名、性别、年龄。每名学生运动员只属于一个学院，一个学院可以有多名学生运动员。
3. 比赛项目信息包括项目编号、项目名称、项目类别、计分单位。一个比赛项目可以安排多场比赛，例如预赛、复赛、决赛。
4. 场地信息包括场地编号、场地名称、位置。每场比赛只能安排在一个场地举行，一个场地可以承办多场比赛。
5. 裁判信息包括裁判编号、姓名、级别。每场比赛必须有一名主裁判，也可以有多名普通裁判；一名裁判可以参与多场比赛。
6. 学生运动员报名参加某场比赛时，需要记录号码、道次、比赛成绩、名次和该学生在该场比赛中为学院获得的积分。一名学生可以参加多场比赛，一场比赛可以有多名学生参加。

请根据以上需求完成如下设计：

（1）构造满足需求的实体联系（ER）模型，要求写出实体、主要属性、联系类型及必要说明。（10 分）

（2）将 ER 模型转换为关系模式，要求写出各关系模式，并标明主码和主要外码。（10 分）

## 四、关系模式规范化题（10 分）

设有关系模式：

```text
R(A, B, C, D, E, F, G, H)
```

R 上的函数依赖集为：

```text
F = { A -> BC, D -> EF, F -> G, AD -> H }
```

请完成：

（1）求 R 的候选码。（2 分）

（2）将 R 逐步分解为满足 BCNF 范式要求的关系模式，要求写出规范化过程。（8 分）

## 五、可选考查题：并发控制与恢复（10 分）

设事务 `T1`、`T2`、`T3` 的一个并发调度 S 如下，其中 `r(X)` 表示读数据项 X，`w(X)` 表示写数据项 X。

| 时刻 | T1 | T2 | T3 |
|---|---|---|---|
| t1 | r(X) |  |  |
| t2 |  |  | r(Z) |
| t3 |  | r(Y) |  |
| t4 | w(X) |  |  |
| t5 |  |  | w(Z) |
| t6 |  | r(Z) |  |
| t7 | r(Y) |  |  |
| t8 |  |  | r(X) |
| t9 | w(Y) |  |  |
| t10 |  |  | w(X) |
| t11 |  |  | rollback |
| t12 |  | commit |  |
| t13 | commit |  |  |

请回答：

（1）画出该调度的优先图，并判断 S 是否为冲突可串行化调度。（4 分）

（2）若事务均采用严格两段锁协议，按该调度的访问顺序执行时是否可能发生死锁？请说明原因。（3 分）

（3）在上述调度中，`T2` 在 `t6` 读取了 `T3` 写入但尚未提交的数据。该问题属于什么问题？可通过哪类并发控制或恢复策略避免？（3 分）

# 数据库系统押题清单

> 基于今年考试说明、2021-2022 真题、2024 回忆卷、2021 回忆卷和 2015 老卷整理。押题不是猜原题，重点是把高频题型练到能稳定落笔。

## 一、必练题型

### 1. 固定学生选课模式上的关系代数和 SQL

高概率继续使用：

```text
Department(dNo, dName, addr, tel)
Student(sNo, sName, sex, age, dNo)
Course(cNo, cName, type, credit, dNo)
CourseRel(cNo, cPNo)
SC(sNo, cNo, score)
```

必须练熟：

- 关系代数：选择、投影、连接、差、除。
- SQL：`JOIN`、`GROUP BY`、`HAVING`、聚合函数、子查询、`CREATE VIEW`、`ALTER TABLE ADD CONSTRAINT`、`UPDATE`。
- 不重点练：SQL 里的并、交、差集合运算；存在谓词。

最像真题的查询方向：

- “选了 A 但没选 B”：关系代数用差；SQL 可用分组或 `NOT IN`。
- “选了全部某类课程”：关系代数用除。
- “课程先修课”：`CourseRel` 自连接 `Course`。
- “不及格/空成绩”：`score < 60 OR score IS NULL`。
- “总学分/平均成绩/选课人数”：`GROUP BY + HAVING`。
- “创建视图”：学生成绩视图、课程统计视图、学院学生平均成绩视图。

### 2. ER 设计与关系模式转换

高概率考一个业务场景，要求：

- 画或描述 ER 模型。
- 写实体、属性、联系和基数。
- 转换为关系模式，标主码和外码。

最容易出题的联系：

- 1:N：学院-学生、图书-副本、项目-比赛、餐桌-服务员。
- M:N：学生-课程、读者-图书预约、运动员-比赛、订单-菜品。
- 带属性的联系：报名成绩、借阅时间、点餐数量、比赛数据。
- 同一实体参与两种角色：比赛中的主队/客队、借阅中的借出馆员/归还馆员。

答题时要写清：

- 关系名和属性。
- 主码。
- 外码参照谁。
- M:N 联系必须转成单独关系。
- 联系自己的属性不能乱放到实体表里。

### 3. BCNF 分解

必练流程：

1. 找不在任何右部的属性，这些属性必须在候选码中。
2. 求属性闭包。
3. 得到候选码。
4. 检查每个非平凡函数依赖左部是否为超码。
5. 用违反 BCNF 的依赖 `X -> Y` 分解：

```text
R1 = X ∪ Y
R2 = R - (Y - X)
```

高频坑：

- 候选码不是看“左边出现多”，而是看闭包能否推出全部属性。
- BCNF 分解不一定保持依赖。
- 分解过程必须写，不要只写最后结果。

## 二、高概率概念题

建议能直接背写：

- 数据模型三要素：数据结构、数据操作、完整性约束。
- 关系模型完整性：实体完整性、参照完整性、用户自定义完整性。
- 数据库设计阶段：需求分析、概念结构设计、逻辑结构设计、物理结构设计。
- 逻辑设计意义：ER 模型转关系模型，确定主码、外码和约束。
- ACID：原子性、一致性、隔离性、持久性。
- 两段锁协议：保证冲突可串行化；严格两段锁避免级联回滚。
- 恢复：提交事务 REDO，未提交事务 UNDO。
- 范式：1NF、2NF、3NF、BCNF 的作用和代价。
- 图数据库语言：Cypher。
- NoSQL：Not Only SQL；MongoDB 集合对应关系数据库表。

## 三、可选考查押题

### 1. 并发控制

可能考：

- 画优先图。
- 判断冲突可串行化。
- 判断是否死锁。
- 判断脏读、不可重复读、丢失修改。

复习重点：

- 冲突边来自不同事务对同一数据项的读写或写写冲突。
- 优先图有环则不是冲突可串行化。
- 两段锁能保证冲突可串行化，但可能死锁。
- 严格两段锁可避免脏读和级联回滚。

### 2. 查询性能分析

可能考：

- 选择、投影下推。
- 先过滤再连接。
- 按连接属性建索引。
- 低基数属性适合位图索引。
- 高选择性条件更值得建 B+ 树索引。

### 3. 新技术

可能考填空或简答：

- 图数据库：节点、边、属性；查询语言 Cypher。
- XML：可扩展标记语言，用于半结构化数据描述与交换。
- NoSQL：键值、文档、列族、图数据库。
- MongoDB：文档、集合、数据库。

## 四、最后复习优先级

1. 先把两套模拟卷的关系代数和 SQL 全部重做一遍，尤其除法、分组聚合、视图、补主外键。
2. 再练 ER 转关系模式，重点检查 M:N 联系有没有单独建表。
3. 最后练 BCNF，要求每一步闭包和分解都能写出来。
4. 考前再背概念题，不要把大段时间花在低概率新技术细节上。
